はじめに

CQCQCQは、アマチュア無線の交信記録（QSO）を管理するためのネイティブアプリです。Mac、iPhone、iPad、Apple Watchに対応し、Apple純正フレームワーク（SwiftUI、SwiftData、AppKit、CoreLocation）のみで構築されています。

サードパーティライブラリやクロスプラットフォームフレームワークは一切使用していないため、各プラットフォームの特性を最大限に活かした、軽量で高速な動作を実現しています。

インストール

App Storeからダウンロードしてインストールできます。Mac、iPhone、iPad、Apple Watchそれぞれに最適化されたユニバーサルアプリです。

QSOの記録

CQCQCQでは、以下のフィールドを使って交信を記録できます。

  • コールサイン — 交信相手のコールサイン
  • RS / RST レポート — 受信・送信のシグナルレポート
  • 周波数 — 使用周波数（コンボボックスでプリセット選択 or 自由入力）
  • モード — CW、SSB、FM、FT8 など
  • 使用機材 — リグ・アンテナなどの情報
  • 送信出力 — TX パワー
  • 開始/終了時刻 — QSOの開始時刻と終了時刻
  • QTH — 相手局の所在地
  • 名前 — 交信相手のオペレータ名
  • 備考 — 自由記述のメモ欄

macOSでは、リスト上のQSOをダブルクリックすると詳細ウィンドウが開きます。

検索・フィルタ

macOS

ツールバーに統合されたFinder風の検索フィールドを使用します。クリックで展開、Escキーで収縮します。コールサイン、名前、QTH、周波数など複数のフィールドを横断検索できます。スコープの切り替えやオートコンプリートにも対応しています。

iOS / iPad

フィルタシートから、使用機材、周波数、モード、期間、QSL状態で絞り込みが可能です。

辞書機能

内蔵辞書として以下のリファレンスを搭載しています。

  • JCC/JCG辞書 — 日本の市（JCC）・郡（JCG）コードを都道府県で絞り込み検索。コードのワンクリックコピーに対応。
  • Qコード辞書 — アマチュア無線で使用される主要なQコードのリファレンス
  • モールス符号表 — 欧文モールスと和文モールス（ワブン）の一覧
  • フォネティックコード — NATOフォネティックアルファベット

モールス変換

テキストをモールス符号に変換し、音声で再生できるユーティリティです。

  • 欧文モールス・和文モールスに対応
  • WPM（Words Per Minute）速度の調整
  • 再生中のカラオケ風ハイライト表示
  • クリップボードへのコピー

macOSでは ⌘⇧M でユーティリティウィンドウとして開けます。iOS/iPadでは設定画面からアクセスできます。

バンドプラン

日本のアマチュアバンドプランを内蔵表示します。周波数帯ごとの使用区分を一覧で確認でき、無線従事者の級別フィルタリングにも対応しています。

macOSでは ⌘⇧B でユーティリティウィンドウとして開けます。

データ入出力

ADIF（Amateur Data Interchange Format）

ADIF形式でのインポート・エクスポートに対応しています。他のロギングソフト（RUMlogNG、QSOMateなど）とのデータ交換が可能です。ADIFファイルはダブルクリックでCQCQCQに直接読み込めます。

CSV

QSOデータのCSVエクスポート・インポートに対応しています。日本国内で広く使われるTurbo HAMLOG形式のCSV取り込みにも対応しています。

設定

以下の項目をカスタマイズできます。

  • 表示する周波数帯の選択
  • 使用モードの選択
  • 使用機材のリスト管理
  • QTH（運用地）のリスト管理
  • QSL送信方法の設定
  • 自局コールサイン・名前の設定
  • 表示項目のカスタマイズ

ビルド方法

ソースコードからビルドする場合は以下の手順で行います。

git clone https://github.com/nobtaka/CQCQCQ.git
cd CQCQCQ
open CQCQCQ.xcodeproj

Xcodeでターゲットプラットフォーム（Mac / iPhone / iPad / Apple Watch）を選択してビルド・実行してください。

FAQ

Q: サードパーティライブラリは使っていますか？

いいえ。CQCQCQはApple純正フレームワーク（SwiftUI、SwiftData、AppKit、CoreLocation、AVFoundation、PencilKit、WatchKit）のみで構築されています。

Q: iCloud同期はできますか？

現在開発中です。CloudKitによるデバイス間データ同期は、ロードマップのフェーズ1として計画されています。

Q: Apple Watch単体で使えますか？

はい。Apple Watch版ではQSOログの一覧・詳細表示、新規QSO入力、スワイプ削除が可能です。

Q: HAMLOG のデータを取り込めますか？

はい。Turbo HAMLOG形式のCSVファイルのインポートに対応しています。

Q: 英語UIは対応していますか？

多言語対応はロードマップに含まれていますが、現時点では日本語UIが中心です。
